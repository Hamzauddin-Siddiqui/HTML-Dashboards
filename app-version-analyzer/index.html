üîç How We Calculate App Quality Metrics

App Quality Cases:
  • Only includes cases with topic tags starting with #app-
  • Excludes #app-samebug and #app-nonavigation(here) as they don't represent actual app functionality issues
  • Focuses on genuine app quality problems that affect user experience
Active Tenants:
  • A tenant is considered "active" for an app version only if they reported 5 or more app quality bugs for that specific version
  • This threshold ensures statistical validity and prevents single-bug outliers from skewing the analysis
  • Provides fair comparison between versions with different deployment scales
Bugs/Day/Tenant Metric:
  • Primary quality metric that normalizes for both time and tenant activity
  • Formula: (App Quality Cases √∑ Operational Days) √∑ Active Tenants
  • Higher values indicate worse app quality relative to deployment scale
  • Enables fair comparison between versions with different lifespans and tenant bases
Operational Days:
  • Calculated from first bug report to last bug report for each app version
  • Formula: Math.floor((endDate - startDate) / (1000 * 3600 * 24)) + 1
  • Minimum of 1 day to avoid division by zero
